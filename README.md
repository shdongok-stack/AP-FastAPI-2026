Сервис для сокращения длинных ссылок.

Сервис позволяет:
- создавать короткие ссылки
- переходить по короткой ссылке
- получать статистику переходов по короткой ссылке
- обновлять и удалять ссылки
- искать ссылки по оригинальной ссылке
- получать список истекших ссылок

API реализовано на _FastAPI_ и использует _PostgreSQL_ в качестве базы данных.\
Для кэширования популярных ссылок используется _Redis_.

------------------------------------------------------------------------

# Описание API

  Метод    Endpoint                    Описание
  -------- --------------------------- ----------------------------------
  POST     /links/shorten              Создание короткой ссылки
  GET      /links/{short_code}         Переход по короткой ссылке
  PUT      /links/{short_code}         Обновление оригинальной ссылки
  DELETE   /links/{short_code}         Удаление ссылки
  GET      /links/{short_code}/stats   Получение статистики по ссылке
  GET      /links/search               Поиск по оригинальной ссылке
  GET      /links/expired              Получение списка истекших ссылок

------------------------------------------------------------------------

# Примеры запросов

## Создание короткой ссылки

POST /links/shorten

``` json
{
  "original_url": "https://example.com",
  "short_name": "example",
  "expiration_date": "2026-04-01T12:00:00"
}
```

Ответ:

``` json
{
  "original_url": "https://example.com",
  "short_name": "example",
  "creation_date": "2026-03-10T10:20:00"
}
```

------------------------------------------------------------------------

## Получение статистики

GET /links/{short_code}/stats

Запрос: example

Пример ответа:

``` json
{
  "original_url": "https://example.com",
  "creation_date": "2026-03-10T10:20:00",
  "click_count": 15,
  "last_used": "2026-03-11T09:00:00"
}
```

# Инструкция по запуску

## 1. Клонировать репозиторий

``` bash
git clone https://github.com/USERNAME/REPOSITORY_NAME.git
cd REPOSITORY_NAME
```

------------------------------------------------------------------------

## 2. Создать файл .env

Создать файл `.env` в корне проекта:

    DB_USER=postgres
    DB_PASS=postgres
    DB_HOST=db
    DB_PORT=5432
    DB_NAME=shortlinker

------------------------------------------------------------------------

## 3. Запустить проект через Docker

``` bash
docker-compose up --build
```

После запуска API будет доступен по адресу:

    http://localhost:8000

Swagger документация:

    http://localhost:8000/docs

------------------------------------------------------------------------

# Описание базы данных

Проект использует PostgreSQL.

Основная таблица:

## links

  Поле              Описание
  ----------------- --------------------------
  id                идентификатор ссылки
  original_url      оригинальная ссылка
  short_name        короткий код
  creation_date     дата создания
  expiration_date   дата истечения
  click_count       количество переходов
  last_used         дата последнего перехода
